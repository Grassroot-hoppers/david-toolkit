# Security Policy

David Toolkit is early-stage software. Treat it as a demo and contributor-ready prototype, not production infrastructure.

## Supported Scope

Right now, security reports are relevant for:

- arbitrary file access through import paths
- script injection via parsed exports
- accidental exposure of private data in sample/demo artifacts
- dependency vulnerabilities that affect the local demo path

## Reporting

Do not open public issues for sensitive security problems.

Email: `staycreative@julien.care`

Include:

- clear reproduction steps
- affected files or commands
- impact level
- whether the issue involves private retail data

## Response Standard

- acknowledgment target: 72 hours
- fix or mitigation target for serious issues: as soon as feasible
